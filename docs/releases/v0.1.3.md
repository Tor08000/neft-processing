# NEFT Processing v0.1.3 — стабильный admin stack

## Overview
Release v0.1.3 bundles everything shipped after v0.1.1: the Pydantic v2 migration and admin-web optimizations from v0.1.2 plus the core-api lifespan upgrade and test hardening from v0.1.3. The admin stack now loads faster, reuses cached queries, and cleanly initializes/tears down backend resources.

## What's included

### Backend
- Pydantic v2 applied to primary admin schemas (limits, merchants, terminals, operations, cards) for cleaner validation and forward compatibility.
- Operations table optimized with composite/partial/BRIN indexes and deterministic ordering by `created_at` + `operation_id`.
- Core API now uses FastAPI lifespan: database and seed data initialize on startup, resources shut down gracefully.
- Hardened Celery configuration: dedicated Redis DBs for broker/result/cache, tuned prefetch and worker recycling, task time limits, Flower behind basic auth.
- Split public/admin API surfaces: `/api/core/api/v1/*` vs `/api/core/api/v1/admin/*`.

### Frontend (admin-web)
- React Query with local stubs and caching for dashboard, operations, billing, clearing, and health pages.
- Lazy routing via `React.lazy` + `Suspense`, reducing the initial bundle by splitting routes.
- Typed table columns with date filters and `date_from`/`date_to` aliases for consistent requests.
- `npm run build` produces a compact Vite bundle (optimized chunks, cache-friendly assets).

### Gateway
- Gzip compression and cache-control for static assets plus balanced proxy timeouts.
- Example production config documented with HTTP/2 + TLS.

### Diagnostics & Tests
- Updated `docs/diag/inspect_neft_repo.py` to drive service-specific test runs.
- Stable pytest runs for core-api, auth-host, ai-service, and workers; admin-web `npm run build` is green.
- Latest diagnostic snapshot: `docs/diag/neft_state_2025-12-03_v0.1.3.txt` (generate via runbook if missing).

## Upgrade notes
- From v0.1.1 to v0.1.3: run core-api migrations (`alembic upgrade head`), verify Redis/Celery/gateway env vars, and rebuild Docker images.
- API contracts are backward-compatible; endpoints were extended/optimized without breaking changes.
