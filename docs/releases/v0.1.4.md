# v0.1.4 – Observability & Reliability

## Highlights
- Added OpenTelemetry collector, Jaeger, Prometheus and Grafana to docker-compose for local observability.
- Introduced environment-aware config scaffolding for core-api, auth-host, ai-service and workers.
- Hardened gateway logging with structured JSON format and upstream timing fields.
- Shipped Grafana dashboards for system overview, business KPIs, Celery health, and error triage.
- Added Alembic safety utilities for checking migration state and dumping schema snapshots.

## Upgrade notes
1. Bring up new tooling with `docker compose up -d otel-collector jaeger prometheus grafana`.
2. Point services at `OTEL_EXPORTER_OTLP_ENDPOINT=http://otel-collector:4317` when enabling tracing.
3. Grafana is pre-provisioned with Prometheus and Jaeger datasources (default admin/admin).
4. Use `python scripts/check_migrations.py` before deployments to ensure DB head alignment.
