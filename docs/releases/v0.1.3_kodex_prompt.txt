## Release description (for GitHub)
```
## NEFT Processing v0.1.3 — стабильный admin stack

### Overview
Bundled v0.1.2 + v0.1.3 improvements over v0.1.1: Pydantic v2 schemas, admin-web caching/lazy routing, and a lifespan-based core-api with clean resource management.

### Backend
- Pydantic v2 for admin schemas (limits, merchants, terminals, operations, cards).
- Operations table indexes (composite/partial/BRIN) and deterministic ordering by created_at + operation_id.
- FastAPI lifespan startup/shutdown for core-api; DB init + graceful teardown.
- Hardened Celery: dedicated Redis DBs, tuned prefetch/limits, Flower with auth.
- Clear public vs admin API namespaces: `/api/core/api/v1/*` and `/api/core/api/v1/admin/*`.

### Frontend (admin-web)
- React Query with local stubs and cached queries across dashboard/operations/billing/clearing/health.
- Lazy routing with `React.lazy` + `Suspense` to split the bundle.
- Typed tables with date filters and `date_from`/`date_to` aliases; optimized Vite build size.

### Gateway
- gzip + cache-control for static assets, balanced proxy timeouts.
- Docs for production TLS/HTTP2 config (gateway/nginx).

### Diagnostics & Tests
- `docs/diag/inspect_neft_repo.py` now drives per-service test runs.
- Stable pytest runs: core-api, auth-host, ai-service, workers; admin-web `npm run build` green.
```
