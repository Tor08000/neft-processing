# NEFT Processing — Release v0.1.1

Milestone **v0.1.1 — Admin UI stable, TS filters synced, gateway stabilized**.

## Overview
This release finalizes the stable integration of the admin interface, gateway routing, and backend operation filtering. The system now provides a fully functional admin UI with consistent API behavior and type-safe frontend requests.

## Highlights
- Admin UI now builds successfully with TypeScript validation
- Shared OperationQuery interface synchronized with backend (date aliases included)
- Gateway proxy rules stabilized for `/admin/` prefix with correct SPA routing
- Backend operations, billing, and clearing endpoints aligned with admin-web expectations
- End-to-end flow tested: Auth → Capture → Billing summary → Clearing batches
- Diagnostic tooling extended with snapshot generation and repo structure validation

## Stability
This version is a safe rollback point for further development of API, database optimization, Celery configuration, and gateway performance improvements.
